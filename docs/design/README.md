# System Design Documentation

This directory contains architectural design documents, system overviews, and high-level design decisions.

## Contents

### Core Architecture
- **ARCHITECTURE.md** - System overview with diagrams, data flow, and component relationships
- **SERVICES.md** - Documentation for all 8 services (Rules Lawyer, Interpreter AI, Data Broker, etc.)
- **STAGES.md** - Pipeline stage definitions and message flow contracts

### Design Decisions
- **assumptions.md** - Project assumptions, constraints, and design rationale

## Purpose

These documents describe:
- How the system is structured
- How components interact
- Design patterns and decisions
- Service responsibilities and interfaces

## For Developers

Start with **ARCHITECTURE.md** for a system overview, then dive into specific services with **SERVICES.md**.

## For AI Agents

These documents provide the foundation for understanding the codebase structure and making informed implementation decisions.
